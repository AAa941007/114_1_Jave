醫院門診預約管理系統使用說明

一、 系統設計架構與概述

本系統是一套基於 Java 語言開發的高階主控台管理軟體，旨在透過結構化開發模式（Specification-Driven Development, SDD）實現醫療資源的數位化管理。系統核心納入了多層繼承架構、模板方法模式（Template Method Pattern）以及遞迴搜索算法，確保在處理病患掛號、醫師排班及病歷檢索時具備高度的準確性與可擴充性。

二、 環境初始化與啟動規範

系統啟動時會自動偵測執行路徑並進行工作目錄宣告。

啟動檢核：系統會即時掃描記憶體中的醫師與病患物件個體，並於主介面頂端顯示當前數據統計。

選單驅動機制：採用迴圈式輸入監聽，確保使用者在各項功能操作完畢後能返回主導覽畫面。

版本控制：當前版本 v2.1 支援更完善的例外處理與 CSV 資料持久化功能。

三、 數據持久化機制：從檔案載入資料

本功能旨在恢復歷史數據，維持醫療業務的連續性。

操作方式：於主選單執行功能編號 11。

執行流程：系統透過緩衝流讀取 patients.csv、doctors.csv 與 appointments.csv 等實體檔案。

關聯重建：讀取過程中，系統會自動將 CSV 的純文字資料轉換為 Java 物件，並根據 ID 重建病患與醫師之間的預約關聯（Mapping）。

匯入反饋：完成後將回傳各類別成功匯入的筆數，若筆數為零則代表檔案不存在或格式不符。

四、 診所資源狀態視覺化

本功能提供管理者宏觀的資源掌握能力，執行編號為 9。

醫師狀態檢視：系統輸出包含醫師姓名、專業學科（Specialty）及服務可用性（isAvailable）。若狀態標註為服務中，代表該醫師可接受排班。

病患縮影清單：列出所有註冊病患的唯一識別碼（ID）及其當前持有的有效預約筆數。此數據是系統執行預約限額檢查的重要依據。

五、 病患註冊與數據唯一性校驗

為了確保醫療行為能準確對應至特定個體，系統對病患註冊（功能編號 1）設有嚴格限制。

屬性登錄：系統要求輸入 ID 與姓名作為強制欄位，電話與電子郵件若未輸入，系統將自動填充預設預留值。

唯一性校驗：若輸入的 ID 與現有病患重複，系統會觸發自定義錯誤訊息告知「Patient with this ID already exists」，防止數據冗餘與後續掛號衝突。

六、 預約掛號標準作業程序 (SOP)

預約掛號（功能編號 2）是系統中最核心的交易邏輯，遵循多階段驗證流程。

身分確認：從動態生成的病患清單中選取預約對象。

資源對應：選取目標醫師後，系統會自動過濾該醫師的可用時間片段（Available Slots）。

類型分類：區分一般門診（Regular）與急診（Emergency）。

自動化執行：系統啟動「預約程序」，依序執行檢查、配置、鎖定時段與寫入資料庫等步驟。

七、 業務規則限制與例外處理

系統內部封裝了強健的業務邏輯，以防止醫療資源濫用。

病患限額限制：每位病患持有之預約數上限為 3 筆。若超過此限，系統在執行預約程序的第一階段（SOP-1）即會攔截並顯示「Patient has reached maximum appointment limit」，以維護掛號公平性。

醫師資源限制：若醫師當日已達掛號負載上限（Daily Workload Limit），系統將自動隱藏可用時段，避免過勞與醫療品質下降。

八、 預約紀錄檢索與計費核算

透過功能編號 4，管理者可以調閱全院所有預約紀錄。

資料整合：輸出內容包含預約流水號、對應病患與醫師、執行時段等。

財務預估：系統會根據預約類型自動呼叫 Billable 介面的計費函數。一般門診會根據掛號費與健保給付比例，計算出預估的「病患自付額」。

急診優先級：急診預約會額外顯示優先權等級與預估等待時間。

九、 病歷管理與遞迴檢索技術

本系統將病歷存儲於病患物件內部的鏈結結構中，執行編號為 7。

紀錄新增：新增診斷內容時，新節點會插入至鏈結頭部，實現「最新紀錄優先」的存取特性。

遞迴搜尋邏輯：使用者輸入關鍵字後，系統會啟動遞迴算法，從最新病歷開始逐筆向後比對診斷字串。此方法能有效處理深度不確定的病歷資料夾。

十、 醫師排班與時段狀態監控

管理醫師時段（功能編號 6）允許系統對所有醫務人員的診表進行完整校閱。

診表確認：系統會模擬診表巡檢流程，輸出醫師當前的排班負荷。

時段動態更新：一旦時段被成功預約，該時段將立即從醫師的可用清單中移除，確保排班數據的即時性與一致性。

十一、 每日營運數據統計

每日統計功能（功能編號 8）提供營運端的量化分析。

產出指標：統計當日全院總掛號人次，包含一般門診與急診。

決策支援：管理者可透過統計數據判斷當日診所負載是否過重，作為調整醫師人力或開放額外時段的依據。

十二、 系統登出與資料同步協議

為了防止數據流失，系統要求使用者透過功能編號 10 安全退出。

自動序列化：在程式結束前，系統會將記憶體中所有對象的狀態同步至外部 CSV 檔案中。

資源釋放：關閉所有檔案串流與 I/O 緩衝區，確保作業系統資源回收完畢，並完整結束主程序。
